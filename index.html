    Areas de git

    Working Directory

    Esta es el area de trabajo donde vamos a crear, editar o eliminar los archivos que formen parte de nuestro proyecto. Muy rara vez estaremos usando la consola en esta instancia. Básicamente estaría representada por la carpeta donde inicializamos el repo Luego de modificar el archivo ponemos: git add . --con el punto indicamos que se adiciona todo. A veces no debe ser así

    Staging area (INDEX)

    Es el area de control de cambios donde se realizan las capturas de código (Snapshot) y basicamente donde indicamos a qué archivos vamos a estar realizando el seguimiento. una vez generados los cambios solamente debemos confirmalos para agregarlos al repositorio. Con git status, veo que está modificado, pero para saber cuales son los cambios, debo ejecutar git diff

    Repository

    Es el area donde se almacenan las confirmacionesde los cambios realizados en archivos de tipo BLOB (binary large Object file), se utilizan como puntos de referencia o control, a los que podemos volver, siempre y cuando indiquemos a GIT cual es el número de Hash o posición que queremos visualizar.

    • git commit abre el editor para realizar la confirmacion de cambios
    • git log muestra el historial de confirmacion de manera descendente
    • git blame <file> muestra el autor, numero de HASH, fecha y hora en la que se modifico cada linea del archivo
    • git checkout <ref> mueve el apuntador a la posicion seleccionada
    • git fetch <remote> captura el histrial de cambios del repositorio remoto
    • git push <remote> <branch sube los cambios de la rama seleccionada al repositorio remoto
